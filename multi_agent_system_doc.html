1. 系统概述

本系统采用 LangChain Handoffs 模式，实现了多个独立 Agent 之间的状态交接和协作。 与传统的 Subagents 模式（将子 Agent 作为工具）不同，Handoffs 模式中每个 Agent 都是独立的节点， 可以主动交接控制权给其他 Agent，同时共享全局状态。

核心特点

  • ✅ 独立 Agent 节点：每个 Agent 是独立节点，不是工具
  • ✅ 状态共享：所有 Agent 共享全局状态（用户信息、上下文等）
  • ✅ 动态路由：每轮对话都重新识别意图（Human-in-the-loop）
  • ✅ 信息补充：支持多轮对话逐步收集缺失信息
  • ✅ 身份校验：所有 Agent 可访问用户信息进行权限验证

架构说明

系统采用 中心化路由 + 分布式处理 的架构：

  • 意图识别 Agent：作为系统的入口，每轮对话都会调用
  • 业务 Agent：三个专门的业务处理 Agent，各自负责不同领域
  • 状态管理器：维护全局状态，所有 Agent 共享访问
  • Handoff 工具：实现 Agent 之间的控制权交接

3. Agent 详解

🎯 意图识别 Agent (intent_recognizer)

职责：识别用户意图，决定路由到哪个业务 Agent

输入：用户消息、对话历史

输出：意图类型、置信度、识别理由、缺失信息列表

支持的意图类型：

  • info_query 信息查询
  • business_handle 业务办理
  • problem_consult 问题咨询
  • unknown 未知意图

特点：

  • 每轮对话都会调用（Human-in-the-loop）
  • 可以检测信息缺失，列出需要补充的信息
  • 识别后自动交接给对应的业务 Agent

📊 信息查询 Agent (info_query_agent)

职责：处理信息查询类问题（位置、规则、数值等）

处理场景：

  • "合同号12345的合同状态是什么？"
  • "报销标准在哪里查看？"
  • "员工信息查询"

能力：

  • 访问用户信息进行身份校验
  • 使用已收集的信息进行查询
  • 请求缺失信息
  • 查询完成后交接回意图识别器

⚙️ 业务办理 Agent (business_handle_agent)

职责：处理操作类业务（签约、提交申请、发起审批等）

处理场景：

  • "帮我给李同学进行签约操作"
  • "提交报销申请"
  • "发起审批流程"
  • "查询候选人张三的面试状态" (Moka 业务，business_type=2)

特殊能力：

  • 身份校验和权限验证
  • 支持 Moka 招聘系统操作（business_type=2）
  • 信息完整性检查
  • 业务操作执行

💡 问题咨询 Agent (problem_consult_agent)

职责：处理问题解决方案类咨询

处理场景：

  • "我的账号报错了该怎么处理？"
  • "如何重置密码？"
  • "系统登录失败怎么解决？"

能力：

  • 问题诊断和分析
  • 提供解决方案建议
  • 请求更多问题细节
  • 访问用户信息辅助诊断

4. 状态管理

系统使用 SharedAgentState 作为全局共享状态，所有 Agent 都可以访问和修改。

状态字段 类型 说明 用途
user_id str 用户ID 身份标识、权限校验
user_name str 用户姓名 个性化回复、日志记录
user_role str 用户角色 权限验证、功能限制
conversation_history List[Dict] 对话历史 上下文理解、多轮对话
current_intent str 当前识别的意图 路由决策、状态跟踪
intent_confidence float 意图识别置信度 质量评估、错误处理
intent_reason str 识别理由 可解释性、调试
missing_info List[str] 缺失信息列表 信息补充、完整性检查
collected_info Dict[str, Any] 已收集的信息 信息复用、业务处理
active_agent str 当前活跃的 Agent 路由控制、状态跟踪
need_intent_check bool 是否需要重新识别意图 Human-in-the-loop 控制

状态流转示例

初始状态

{ "user_id": "U001", "user_name": "张三", "user_role": "员工", "conversation_history": [], "active_agent": "intent_recognizer", "need_intent_check": true }

意图识别后

{ "current_intent": "business_handle", "intent_confidence": 0.95, "intent_reason": "包含'进行签约操作'动作词", "active_agent": "business_handle_agent", "need_intent_check": false }

5. Handoff 机制

Handoff 是 Agent 之间交接控制权的机制。通过 handoff_to_agent 工具实现。

Handoff 流程

步骤 1：当前 Agent 决定需要交接
步骤 2：调用 handoff_to_agent(target_agent, reason)
步骤 3：更新状态中的 active_agent
步骤 4：如果交接回意图识别器，设置 need_intent_check = true
步骤 5：下一轮对话由目标 Agent 处理

Handoff 场景

  • 意图识别 → 业务 Agent：识别意图后自动交接
  • 业务 Agent → 意图识别：业务完成后交接，等待下一轮
  • 业务 Agent → 业务 Agent：需要其他 Agent 协助时交接

6. 完整流程

单轮对话流程

1. 用户输入
用户发送消息："帮我查询合同号12345的合同状态"
2. 更新对话历史
将用户消息添加到 conversation_history
3. 检查是否需要识别意图
检查 need_intent_check 或 active_agent == "intent_recognizer"
4. 调用意图识别 Agent
输入：用户消息 + 对话历史（最近3轮）
输出：意图类型、置信度、理由
5. 更新状态
设置 current_intent、intent_confidence、intent_reason
6. 路由决策
根据意图类型选择目标 Agent：
  • info_query → info_query_agent
  • business_handle → business_handle_agent
  • problem_consult → problem_consult_agent
7. 调用业务 Agent
目标 Agent 处理业务逻辑，可能需要：
  • 访问用户信息进行身份校验
  • 使用已收集的信息
  • 请求缺失信息
  • 调用业务工具（如 Moka 查询）
8. 业务 Agent 完成
调用 handoff_to_agent("intent_recognizer", "业务完成")
9. 返回结果
将 Agent 响应返回给用户，更新对话历史
10. 等待下一轮
状态设置为 active_agent = "intent_recognizer"，need_intent_check = true

多轮对话流程（信息补充场景）

轮次 1：用户："帮我给李同学进行签约操作"
→ 意图识别：business_handle
→ 业务 Agent 检查：缺少合同号
→ 请求缺失信息："需要补充信息：合同号"
轮次 2：用户："合同号是12345"
→ 意图识别：business_handle（重新识别）
→ 业务 Agent：保存信息 save_collected_info("合同号", "12345")
→ 执行业务操作：签约成功
轮次 3：用户："查询候选人张三的状态"
→ 意图识别：business_handle（Moka 业务）
→ 业务 Agent：调用 moka_query_candidate("张三")
→ 返回 Moka 查询结果

7. 工具说明

Handoff 工具

  • handoff_to_agent(target_agent, reason)
    将控制权交接给指定的 Agent。用于在意图识别后路由到对应的业务 Agent，或在不同 Agent 之间切换。

信息管理工具

  • request_missing_info(info_needed)
    当业务处理需要更多信息时，向用户请求缺失的信息。会将信息添加到 missing_info 列表。
  • save_collected_info(key, value)
    保存从用户那里收集到的信息到 collected_info 字典。如果该信息在缺失列表中，会自动移除。

业务工具（Moka 示例）

  • moka_query_candidate(candidate_name)
    查询 Moka 招聘系统中的候选人信息。需要候选人姓名或ID。
    用于 business_type=2 的情况

8. 使用示例

初始化系统

from test2 import MultiAgentSystem # 初始化多 Agent 系统 system = MultiAgentSystem() # 初始化状态（设置用户信息用于身份校验） initial_state = { "user_id": "U001", "user_name": "张三", "user_role": "员工", "conversation_history": [], "active_agent": "intent_recognizer", "need_intent_check": True, "missing_info": [], "collected_info": {} }

处理用户消息

# 处理用户消息（每轮都会重新识别意图） result = system.process_message("帮我查询合同号12345的合同状态", initial_state) # 获取响应 print(result["response"]) # 获取更新后的状态 updated_state = result["state"] print(f"当前意图：{updated_state['current_intent']}") print(f"活跃 Agent：{updated_state['active_agent']}")

多轮对话示例

current_state = initial_state.copy() # 第一轮：信息查询 result1 = system.process_message("帮我查询合同状态", current_state) current_state = result1["state"] print(result1["response"]) # 第二轮：补充信息 result2 = system.process_message("合同号是12345", current_state) current_state = result2["state"] print(result2["response"]) # 第三轮：Moka 业务（business_type=2） result3 = system.process_message("查询候选人张三的面试状态", current_state) current_state = result3["state"] print(result3["response"])

Moka 业务处理（business_type=2）

场景：查询候选人信息

用户输入："帮我查询候选人张三的面试状态"

流程：

  1. 意图识别 Agent 识别为 business_handle
  2. 路由到 business_handle_agent
  3. 业务 Agent 识别出是 Moka 相关操作
  4. 调用 moka_query_candidate("张三") 工具
  5. 返回 Moka 系统中的候选人信息
  6. 交接回意图识别器，等待下一轮
